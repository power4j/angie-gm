# 正式文档入口

`docs/public/` 存放最终交付、用户可见、可稳定复用的正式文档。

本地过程记录统一放在 `docs/local/`，该目录不纳入 Git 跟踪。需要长期复用或对外引用的内容，应整理后迁入 `docs/public/`。

## 目录说明

- `architecture/`
  - 仓库架构、目录规范、安装布局、edition 设计
- `packaging/`
  - 打包策略、依赖策略、互斥升级规则、安装诊断要求
- `release/`
  - GitHub Actions、GitHub Release、版本发布规则
- `validation/`
  - 验证矩阵、平台兼容性口径
- `reference/`
  - 上游来源、版本基线、补丁、checksums
