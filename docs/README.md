# 文档总览

`docs/` 用于组织本仓库的文档内容，当前分为 `public` 与 `local` 两类。

## 目录结构

- `docs/public/`
  - 正式文档。
  - 面向最终交付、用户可见与外部引用。
- `docs/local/`
  - 本地过程文档。
  - 面向开发过程记录、阶段计划与进度跟踪。

## 术语口径

- 发布：指代码或产物发布到 GitHub，包括 push、merge、tag、GitHub Release。
- 部署：指安装包被安装到目标 Linux 服务器，并完成服务启动、自检与可用性确认。
